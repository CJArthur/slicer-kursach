# Slicer — субтитры и нарезка видео

Десктопное приложение на Python: выбираете видео, папку для сохранения, стиль субтитров и (по желанию) положение текста на кадре. Программа вытаскивает звук, прогоняет распознавание через **WhisperX**, накладывает слова на ролик через **MoviePy**, а если включить галочку — режет готовый файл на равные части **ffmpeg**’ом.

## Что нужно на машине

- **Python 3.11+**
- **ffmpeg** и **ffprobe** в `PATH` (нужны для нарезки и для части пайплайна).
- Шрифты лежат в `app/fonts/` — пути к ним прописаны в `app/presets.py`.

Зависимости ставятся из корня:

```bash
pip install -r requirements.txt
```

Первый запуск может долго качать модели Whisper — это нормально тк весит пару гигабайт.
Если модель не подходит, то ее можно поменять в файле `app/services/video_slicer.py` в функции transcribe_with_whisperx.

## Как запускать

Рабочая директория должна быть **`app`**, чтобы импорты `services` и `presets` находились:

```bash
cd app
python main.py
```

Итоговое видео: `<выбранная_папка>/<имя_исходника>_subtitled.mp4`.  
Если включена нарезка — куски в `<та_же_папка>/<имя>_parts/<имя>_01.mp4` и дальше по номеру.

## Структура (кратко)

- `app/main.py` — старт UI и порядок шагов обработки.
- `app/services/window.py` — окно и кнопки.
- `app/services/video_slicer.py` — аудио, WhisperX, накладывание субтитров, нарезка.
- `app/presets.py` — цвета/шрифты пресетов.

## Замечания

- Тяжёлые модели и **CPU** дают долгую транскрипцию — окно не должно висеть, обработка идёт в фоне.
- Если что-то ругается на версии `torch` / `pyannote` — это отдельная история окружения, не логики приложения.
